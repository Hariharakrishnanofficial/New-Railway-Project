🚀 POSTMAN SETUP (5 STEPS)
════════════════════════════════════════════════════════════════════════════════

1️⃣  Create new request (+ button)

2️⃣  Method: POST

3️⃣  URL: http://localhost:3000/server/catalyst_backend/api/auth/register

4️⃣  Headers tab → Add:
    Content-Type: application/json

5️⃣  Body tab → Select "raw" → Select "JSON" → Paste sample user


✅ EXPECTED SUCCESS RESPONSE (Status 201)
════════════════════════════════════════════════════════════════════════════════

{
  "success": true,
  "status": "created",
  "data": {
    "message": "User registered successfully",
    "user_id": "123456789",
    "email": "john.doe@railway.com",
    "role": "User"
  }
}


❌ POSSIBLE ERROR RESPONSES
════════════════════════════════════════════════════════════════════════════════

Email already registered (409):
  "details": "Email already registered"
  → Use different email

Invalid JSON (400):
  "error": "Invalid request: JSON body required"
  → Check Body is set to "raw" JSON

Missing fields (400):
  "error": "Missing required fields: Full_Name, ..."
  → Add all required fields


⏭️  NEXT: TEST SIGNIN
════════════════════════════════════════════════════════════════════════════════

After registration succeeds, test signin:

URL: http://localhost:3000/server/catalyst_backend/api/auth/signin
Method: POST
Body:
{
  "Email": "john.doe@railway.com",
  "Password": "Password123!"
}

Response will contain:
  • access_token (use for protected endpoints)
  • refresh_token
  • user data


🔐 USING ACCESS TOKEN
════════════════════════════════════════════════════════════════════════════════

Copy access_token from signin response

Use in Authorization header:
  Authorization: Bearer <access_token>

Example - Get Profile:
  URL: /api/auth/profile
  Headers: Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGc...
